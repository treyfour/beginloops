## Suggested next step
Add a cron-triggered heartbeat endpoint. Create `app/api/cron/heartbeat/route.ts`:

```ts
// app/api/cron/heartbeat/route.ts
import { NextResponse } from 'next/server';

export async function GET() {
  return NextResponse.json({
    status: 'ok',
    timestamp: new Date().toISOString(),
  });
}
```

Then wire it to a schedule by adding `vercel.json` at the repo root:

```json
{
  "crons": [
    {
      "path": "/api/cron/heartbeat",
      "schedule": "*/5 * * * *"
    }
  ]
}
```

This teaches you two things at once: how cron jobs work in a serverless environment (Vercel calls your route on a schedule — there's no persistent process keeping time), and how to build a **heartbeat** endpoint that monitoring tools can ping to confirm your app is alive. It's the foundation of health-check patterns used in production systems everywhere.
