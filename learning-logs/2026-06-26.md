## Suggested next step
Add HMAC signature verification to `app/api/webhooks/route.ts` — this is the security layer every production webhook handler needs. Right now your endpoint accepts any POST request; GitHub signs each delivery with a shared secret using SHA-256, and without this check anyone who knows the URL can send fake events.

Add this helper and update the handler:

```ts
// app/api/webhooks/route.ts
import crypto from "crypto";

function verifySignature(payload: string, signature: string | null, secret: string): boolean {
  if (!signature) return false;
  const expected = `sha256=${crypto.createHmac("sha256", secret).update(payload).digest("hex")}`;
  return crypto.timingSafeEqual(Buffer.from(expected), Buffer.from(signature));
}

export async function POST(req: NextRequest) {
  const rawBody = await req.text();
  const sig = req.headers.get("x-hub-signature-256");
  if (!verifySignature(rawBody, sig, process.env.WEBHOOK_SECRET!)) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }
  const body = JSON.parse(rawBody);
  // ...rest of handler unchanged
}
```

Set `WEBHOOK_SECRET` in both your Vercel environment variables and in the GitHub webhook settings (Settings → Webhooks → Secret). **Why `timingSafeEqual` matters:** using `===` instead would expose a timing side-channel — an attacker could measure microsecond response differences to guess the secret one byte at a time. `timingSafeEqual` runs in constant time regardless of where the strings diverge.
